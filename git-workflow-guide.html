01

Branch Structure

prod

Live production code. Only stable, fully tested features land here.

Managed by TL only · Direct push blocked
dev

Integration branch. All completed features merge here first.

All devs via Pull Request
feature/*

Your personal workspace. Work here for days freely — push anytime.

Individual dev · push freely
hotfix/*

Urgent prod fixes. Branches off prod, merges into prod AND dev.

TL only · fast-track
💡
The Core Rule

Never work directly on dev or prod. Every change — even a one-liner — gets its own branch. feature/* is your safe zone. Push every day.

02

Developer Daily Flow

Follow these steps every time you start a new feature or task. Features can take days — that is expected.

1
Create Your Feature Branch

Always branch off dev. Pull latest first.

git checkout dev
git pull origin dev
git checkout -b feature/your-feature-name
2
Work and Push Every Day

Push to your feature branch daily. Half-done work is fine here.

git add .
git commit -m "wip: recipient comparison logic"
git push origin feature/your-feature-name
✅

Pushing to feature/* does NOT affect dev or prod. You are safe to push unfinished work here every day.

3
Sync With Dev Daily

If other devs merged into dev while you are working, rebase to avoid big conflicts at the end.

git fetch origin
git rebase origin/dev
# fix any conflicts, then:
git push --force-with-lease origin feature/your-feature-name
4
Raise PR When Feature Is Done

When complete and tested, open a Pull Request on GitHub into dev.

📋
PR Checklist

Base: dev  ·  Compare: feature/your-name  ·  Squash wip commits  ·  Add screenshots if UI  ·  Request review

⚠️

Never merge your own PR without at least one review.

5
TL Merges dev → prod

When dev is stable, TL raises a PR from dev to prod and releases.

03

Working in Parallel

Sometimes a hotfix or urgent task interrupts your current feature work. Here are the three scenarios and how to handle each.

Scenario A
Urgent fix needed — feature already pushed
▼

Your code is already pushed to your feature branch. Just switch branches — nothing to stash.

# Your feature is pushed — safe to switch
git checkout prod
git checkout -b hotfix/urgent-fix

# fix the issue, push, raise PR
git commit -m "fix: invoice calculation error"
git push origin hotfix/urgent-fix

# come back to your feature — everything exactly as you left it
git checkout feature/your-feature-name
Scenario B
Work not pushed yet — use stash
▼

If you have uncommitted local changes and need to switch urgently, stash them temporarily.

git stash push -m "wip: halfway through recipient logic"
git checkout dev
git checkout -b feature/urgent-small-thing

# do the urgent work...

# come back and restore
git checkout feature/your-feature-name
git stash pop
04

Rebase

Rebase keeps history clean and linear. Preferred over merge for feature branches.

Basic Rebase — Sync With dev

git checkout feature/your-feature-name
git rebase origin/dev

# if conflicts appear:
# 1. fix the conflict in the file
git add .
git rebase --continue

# or cancel entirely
git rebase --abort

# push after rebase (force needed — history changed)
git push --force-with-lease origin feature/your-feature-name

Interactive Rebase — Clean Up wip Commits

Before raising a PR, squash all your daily wip commits into one clean commit.

git rebase -i origin/dev

# Editor opens. Change 'pick' to 'squash' for wip commits:
pick   abc1234  feat: start recipient logic
squash def5678  wip: halfway done
squash ghi9012  wip: almost there
squash jkl3456  wip: done

# Result: one clean commit instead of four messy ones
🔑
Golden Rule of Rebase

Never rebase a branch others are also working on. Since each dev owns their own feature/* branch, you are always safe to rebase it. After rebase, always use --force-with-lease not --force.

05

Commit Message Convention

Consistent commit messages make history readable for everyone.

feat:
New feature or functionality
feat: add recipient comparison logic
fix:
Bug fix
fix: pdf preview not loading on mobile
wip:
Daily push on feature branch — work in progress
wip: halfway through esign flow
refactor:
Code cleanup, no behavior change
refactor: simplify deepClone logic
chore:
Config, deps, non-code changes
chore: update angular to v17
docs:
Documentation only
docs: update readme with setup steps
⚠️
Before Merging

Squash all wip: commits into one clean feat: or fix: commit using interactive rebase before your PR merges into dev.
